## Summary

The Ondilo ICO Spa V2 case study applies the IoT Integrator agent
(`docs/prompts/iot-integrator-prompt.md`) to a connected pool/spa water
analyzer. It is the first run of the self-augmenting IoT Integrator stage
(consumes the prior three case-study reports as methodological input) and
the lighter of the two new IoT-integrator cases.

- **Outcome shape:** configuration-only Phase 3 (no parallel
  `custom_components/<slug>/`); the upstream HA core integration
  `ondilo_ico` already satisfies the read-only intake.
- **Authentication shape:** clean OAuth2 Authorization Code at
  `interop.ondilo.com/oauth2/{authorize,token}`, public `client_id`
  `customer_api`, empty client secret, single `api` scope, 5 req/s and
  30 req/h per user, 1-hour access tokens, non-expiring refresh tokens.
- **Obscurity-vs-authentication position:** *cloud-openness* counterweight
  to Spider Farmer; same agent, same Technique Inventory, very different
  attacker-mirror size.
