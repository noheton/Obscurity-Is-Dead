Three bugs were identified from the log and fixed:

Bug 1 — Race condition (root cause of most failures): Lines 997–999 show two simultaneous BLE write failed (attempt 1) messages ~1ms apart. This proves the poll loop and an entity service call were both inside _send() at the same time, sharing _pending_id/_resp_event. The result: responses get matched to the wrong caller and commands silently fail. Fixed by wrapping the entire send-wait loop in asyncio.Lock.

Bug 2 — Retry after disconnect: When write_gatt_char raised a connection error (GATT 133, [Errno 104], EOF), the retry loop continued unconditionally. The next attempt then hit "Characteristic 0000ff02 was not found!" because BleakClient was already disconnected. Fixed by checking self.connected after a write exception and breaking immediately if the link is gone.

Bug 3 — Stale HA state after commands: After sending setLight/setFan/setBlower/setHeater the entity state in HA wasn't updated until the next getDevSta poll (30s interval). Commands appeared to "not work" even when the device accepted them (code 200). Fixed by calling coordinator.update() with the commanded values when send_command returns a non-None response.
