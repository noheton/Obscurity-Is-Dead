# Obscurity Is Dead — provenance graph viewer

This directory holds a static, browser-viewable rendering of
`../provenance.ttl` (the W3C PROV-O graph emitted by the Modeler agent,
stage 7 — see `../prompts/modeler-prompt.md`). The graph itself is
regenerated by `scripts/build_provenance_graph.py`; this viewer is
regenerated from the graph by `scripts/build_provenance_site.py`. The
GitHub Actions workflow at `.github/workflows/pages.yml` runs both on
every push to `main` and deploys the result to GitHub Pages.

- **Rule 14 (CLAUDE.md):** deployment of THIS site (the
  provenance-graph viewer) to GitHub Pages was explicitly consented to
  by the human author (Florian Krebs) on 2026-05-04 — recorded in
  `../logbook.md` (Q14 entry). Consent applies to the provenance-graph
  viewer only; public-mirror push of the full repository, Zenodo
  deposit, and arXiv submission remain separate consent gates.
- **Rule 13 (CLAUDE.md):** both the graph build and the site build
  refuse to write their outputs if any literal matches a known
  live-credential pattern. Redaction markers (`[REDACTED:...]`) pass
  through verbatim.

## Build

```
pip install rdflib
python3 scripts/build_provenance_graph.py    # regenerates docs/provenance.ttl
python3 scripts/build_provenance_site.py     # regenerates docs/site/
```

Then open `docs/site/index.html` in a browser, or visit the deployed
GitHub Pages URL configured by `.github/workflows/pages.yml`. Cytoscape.js is loaded
from a CDN; the file works offline once the bundle is cached, or can be
made hermetic by vendoring `cytoscape.min.js` and
`cytoscape-cose-bilkent.js` next to `index.html`.

## What you see

- **Nodes** — every `prov:Entity` / `prov:Activity` / `prov:Agent`
  in the graph, coloured by class (Claim / Source / Activity / Agent /
  Figure / Build / Commit / VerificationStatus).
- **Edges** — `prov:wasDerivedFrom`, `prov:wasGeneratedBy`,
  `prov:wasAttributedTo`, `prov:used`, `prov:wasAssociatedWith`,
  `prov:wasInformedBy`, `prov:generated`, `oid:hasVerificationStatus`.
- **Filters** — class chips, verification-rung chips, free-text search.
- **Inspector** — click a node to see its full triple list.
