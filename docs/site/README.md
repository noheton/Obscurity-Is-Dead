# Obscurity Is Dead — provenance graph viewer (local artifact)

This directory holds a static, browser-viewable rendering of
`../provenance.ttl` (the W3C PROV-O graph emitted by the Modeler agent,
stage 7 — see `../prompts/modeler-prompt.md`). It is regenerated by
`scripts/build_provenance_site.py` and **NOT committed in rendered
form** as a default; treat it as the same kind of artifact as
`paper/main.pdf`:

- **Rule 14 (CLAUDE.md):** the site is a *local* artifact. Deployment
  to GitHub Pages, GitLab Pages, a Zenodo record, or any other public
  surface requires explicit written consent from the human author
  (Florian Krebs). The build script never deploys.
- **Rule 13 (CLAUDE.md):** the build script refuses to write the site
  if any node label, comment, or literal triple matches a known
  live-credential pattern. Redaction markers (`[REDACTED:...]`) pass
  through verbatim.

## Build

```
pip install rdflib
python3 scripts/build_provenance_site.py
```

Then open `docs/site/index.html` in a browser. Cytoscape.js is loaded
from a CDN; the file works offline once the bundle is cached, or can be
made hermetic by vendoring `cytoscape.min.js` and
`cytoscape-cose-bilkent.js` next to `index.html`.

## What you see

- **Nodes** — every `prov:Entity` / `prov:Activity` / `prov:Agent`
  in the graph, coloured by class (Claim / Source / Activity / Agent /
  Figure / Build / Commit / VerificationStatus).
- **Edges** — `prov:wasDerivedFrom`, `prov:wasGeneratedBy`,
  `prov:wasAttributedTo`, `prov:used`, `prov:wasAssociatedWith`,
  `prov:wasInformedBy`, `prov:generated`, `oid:hasVerificationStatus`.
- **Filters** — class chips, verification-rung chips, free-text search.
- **Inspector** — click a node to see its full triple list.
