# `docs/site/` — public-facing GitHub Pages surface

This directory holds the **deployable** website for the
*Obscurity Is Dead* project. It is the canonical
*between-README-and-paper* surface that an external reader
encounters when they visit the project's GitHub Pages deployment.

The Site Agent (stage 8 of the agent pipeline; prompt at
`docs/prompts/site-agent-prompt.md`) owns this directory.

## Deployment (rule 14)

Deployment to GitHub Pages happens via `.github/workflows/pages.yml`,
which runs on push to `main` (the publish branch declared in
`docs/publication-consent.md`). The workflow:

1. **`consent-gate`** — verifies `docs/publication-consent.md` exists,
   names `docs/site/` as authorised, and names `main` as authorised;
   aborts the workflow on any failure.
2. **`build`** — re-runs `scripts/build_provenance_site.py` to
   refresh `graph.html` / `graph.json` from the latest
   `docs/provenance.ttl`; syncs the explicit nine-figure list from
   `paper/figures/` into `docs/site/figures/`; runs a rule-13 grep
   against the rendered site for known live-credential patterns;
   uploads `docs/site/` as the Pages artifact.
3. **`deploy`** — `actions/deploy-pages@v4` to the `github-pages`
   environment.

The Site Agent does **not** trigger the workflow. The human author
triggers it (rule 14) by pushing to `main` under the recorded
consent. A failed consent check causes the workflow to abort *before*
deployment.

## Build (local)

```
pip install rdflib
python3 scripts/build_provenance_graph.py   # regenerates docs/provenance.ttl
python3 scripts/build_provenance_site.py    # regenerates docs/site/graph.{html,json}
# The hand-authored landing pages (index.html, paper.html,
# methodology.html, governance.html) are not regenerated by the
# script; the Site Agent edits them directly per its prompt.
```

Then open `docs/site/index.html` in a browser, or visit the deployed
GitHub Pages URL configured by `.github/workflows/pages.yml`.
Cytoscape.js is loaded from a CDN; the graph viewer works offline
once the bundle is cached, or can be made hermetic by vendoring
`cytoscape.min.js` and `cytoscape-cose-bilkent.js` next to
`graph.html`.

## Rules in force

- **Rule 13 (CLAUDE.md):** the Site Agent runs a pre-emit grep over
  every file it writes; live-credential patterns (AWS keys, OpenSSH
  private keys, JWT tokens, Slack tokens, GitHub PATs) abort the
  pass. Redaction markers (`[REDACTED:...]`) pass through verbatim.
  The Pages workflow runs the same grep against the rendered site
  before deployment.
- **Rule 14 (CLAUDE.md):** the deployable site is rendered locally
  on every Site Agent pass, but *deployment* requires explicit
  written consent from the human author, scoped at
  `docs/publication-consent.md`. Other rule-14 gates (public-mirror
  push of the full repository, Zenodo deposit, arXiv submission)
  remain separate consent records.
- **Rule 16 (CLAUDE.md):** when `README.md` changes a headline KPI,
  the site is re-rendered in the same commit.
- **Rule 17 (CLAUDE.md):** the *Read the paper* CTA points first to
  `paper/main-condensed.pdf` (core submission); the long-form PDF
  is a second-position enrichment link.
- **Rule 18 (CLAUDE.md):** the site is the human-readable rendering
  of the same traceability spine the Aligner audits in prose
  (stage 6) and the Modeler renders as PROV-O (stage 7,
  `docs/provenance.ttl` → `graph.html`).
- **Rule 19 (CLAUDE.md):** every consistency-bearing claim, KPI,
  figure, or citation surfaced on the site has a one-step pointer
  back to its canonical source (paper / README / sources / fair /
  CLAUDE.md). The site never introduces a new claim.
