### Scope — what is covered

The recorded consent covers, and only covers, the following:

1. The hand-authored landing pages owned by the Site Agent:
   `docs/site/index.html`, `paper.html`, `methodology.html`,
   `governance.html`. Every claim, KPI, figure, and citation on
   these pages must trace back to a canonical source the Aligner
   already approves (rule 19 in `CLAUDE.md`).
2. The Cytoscape.js graph viewer at `docs/site/graph.html` and its
   data file `docs/site/graph.json`, generated by

3. The shared stylesheet at `docs/site/style.css`.
4. The figures referenced by the landing pages, copied verbatim
   from `paper/figures/*.svg` into `docs/site/figures/` (no edits).
5. The directory README at `docs/site/README.md`.

### Scope — what is *not* covered

The recorded consent does **not** authorise:

- Publication of the long-form paper PDF (`paper/main.pdf`) or the
  condensed paper PDF (`paper/main-condensed.pdf`) on any public
  surface beyond `docs/site/`'s links to the LaTeX sources in the
  GitHub repository. The site links to the canonical LaTeX sources
  on github.com; it does not host, serve, or mirror the rendered
  PDFs itself.
- Submission to arXiv, Zenodo, or any indexed preprint / archive
  service. The CI build-paper workflow continues to produce draft
  artifacts only; the `make arxiv` Makefile target remains gated
  on a separate, future consent record.
- A public-mirror push of `noheton/spider_farmer` or
  `noheton/powerocean-dev` (the upstream experimental repositories

  upstreams still hold their pre-redaction history and require
  their own `git-filter-repo` passes against the catalogue at
  `docs/redaction-policy.md` before any publication consent can
  cover them.
- Any other surface or any expansion of scope. New surfaces require
  a new entry below.

### Revocation

This consent is revocable at any time by the human author. To
revoke:

1. Append a *Revocation record* below identifying this consent
   record by date and giving the revocation reason.
2. Disable the GitHub Pages source in the repository settings, or
   delete the `.github/workflows/pages.yml` workflow.
3. The Site Agent will continue to render `docs/site/` locally on
   subsequent passes (the directory is part of the research
   record); only *deployment* to a public surface is gated on
   consent.

Revocation invalidates the public surface but not the recorded
artifact. The `docs/site/` directory under git history remains
part of the research record (rule 4); it is not retracted.
